# Examples

## 示例 1：生成提交信息（简单修复）

输入摘要：
- 变更：修复登录超时后未跳转登录页
- 影响范围：`auth` 模块

期望输出：
```text
fix(auth): 修复登录超时后未跳转登录页
```

## 示例 2：生成提交信息（多点改动）

输入摘要：
- 变更：新增导出报表按钮、补充导出接口参数校验、更新文档
- 影响范围：`report` 模块

期望输出：
```text
feat(report): 增加报表导出能力并补充参数校验

新增导出入口并完善服务端参数校验，避免非法条件触发空文件导出。
```

## 示例 3：代码审查（发现问题）

输入摘要：
- diff 显示：循环内重复请求数据库，且无缓存
- 测试：仅覆盖成功路径，无异常路径

期望输出要点：
- 先给整体质量概览
- 问题表格至少包含：
  - 性能问题：循环内重复查询
  - 测试缺口：缺少异常分支测试
- 若无可靠行号，行号写 `N/A`

## 示例 4：代码审查（无阻断问题）

输入摘要：
- diff 为小范围重构，逻辑等价
- 关键路径测试通过

期望输出要点：
- 明确写“未发现阻断问题”
- 说明残余风险（例如仅手工验证，建议补充自动化用例）
